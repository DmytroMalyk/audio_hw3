# Лабораторна робота: Система розпізнавання мовлення на базі Whisper (Toronto Dataset)

---

## 1. Мета роботи

Метою даної лабораторної роботи є побудова системи автоматичного розпізнавання українського мовлення (ASR) шляхом дотренування попередньо навченої моделі Whisper від OpenAI на наборі даних Toronto. Для реалізації навчальної логіки використовується фреймворк PyTorch Lightning, а якість результатів оцінюється за допомогою метрик WER та CER.

---

## 2. Набір даних

### 2.1 Опис

Для виконання роботи використовувався **Toronto Ukrainian Speech Dataset** — набір даних з українського мовлення, що містить аудіозаписи у форматі WAV та відповідні текстові транскрипції. Датасет завантажено з Kaggle (`dmytromalykpn/toronto-data`).

Загальна кількість зразків: **18 303**  
Середня тривалість одного запису: **~6.12 с** (std ≈ 1.03 с, max ≈ 8.38 с)  
Середня кількість слів в одному висловлюванні: **~14.1**  
Середня кількість символів в одному висловлюванні: **~92.1**

## 3. Вибір моделі

Для задачі дотренування обрано **`openai/whisper-small`** з бібліотеки HuggingFace Transformers. Це рішення обумовлено наступними міркуваннями:

- Модель `whisper-small` містить ~244M параметрів, що забезпечує хороший баланс між якістю розпізнавання та швидкістю навчання на одній GPU (Tesla P100 16GB).
- Whisper підтримує українську мову з коробки (`language='ukrainian'`, `task='transcribe'`), що значно спрощує налаштування токенізатора та логіку генерації.
- Більші моделі (`whisper-medium`, `whisper-large`) хоча і показують вищу точність, потребують суттєво більше часу на тренування та значно більшого обсягу GPU-пам'яті.

При завантаженні моделі одразу виставляється `generation_config.forced_decoder_ids` — це примусово спрямовує декодер на транскрипцію українською мовою.

## 4. Архітектура рішення

### 4.1 Обробка даних

На початку роботи всі аудіофайли попередньо завантажуються в оперативну пам'ять (RAM preloading) за допомогою `tqdm`-циклу. Це дозволяє уникнути затримок введення-виведення під час навчання та прискорює обробку батчів на GPU. Зразки, що не вдалося зчитати, пропускаються з попередженням.

Реалізовано два класи:

- **`TorontoDataset`** (успадкований від `torch.utils.data.Dataset`) — зчитує попередньо завантажений масив аудіо (`audio_array`) безпосередньо з пам'яті, виконує нормалізацію тексту та перетворення аудіо у вхідні ознаки (mel-спектрограму) через `WhisperProcessor`.
- **`TorontoDataModule`** (успадкований від `pl.LightningDataModule`) — обгортає три сплітів в окремі DataLoader-и з `num_workers=4` та однаковим `collate_fn`, що виконує padding та маскування міток.

Після завантаження всього датасету в RAM статистики тривалості та кількості слів обчислюються безпосередньо з масивів `audio_array` (без повторного зчитування з диску). Зберігаються також розширені метадані: кількість унікальних слів, символів та загальна кількість токенів у вибірці, які записуються у файл `outputs/eda_stats.json`.

Функція `normalize_text` переводить текст у нижній регістр та видаляє всі символи, крім літер, пробілів та апострофа — це важливо для коректного підрахунку WER/CER.

### 4.2 Модуль навчання

**`WhisperLightningModule`** реалізує повний цикл навчання на основі `pl.LightningModule`:

- **Заморозка енкодера**: енкодер Whisper заморожено (`requires_grad = False`), дотренування виконується лише для декодера. Це дозволяє скоротити кількість навчуваних параметрів до **153M** (з 241M загалом), прискорити навчання та зменшити ризик перенавчання.
- **Оптимізатор**: `AdamW` з розділеним weight decay (не застосовується до bias і LayerNorm).
- **Scheduler**: лінійне затухання з warmup (`get_linear_schedule_with_warmup`).
- **Метрики**: WER та CER обчислюються на валідаційному та тестовому кроках за допомогою бібліотеки `evaluate`.

### 4.3 Конфігурація навчання

| Параметр             | Значення              |
|----------------------|-----------------------|
| Модель               | `openai/whisper-small`|
| Batch size           | 32                    |
| Max epochs           | 8                     |
| Learning rate        | 5e-5                  |
| Warmup steps         | 100                   |
| Weight decay         | 0.01                  |
| Gradient clip        | 1.0                   |
| Precision            | `16-mixed` (fp16)     |
| Early stopping       | patience=2 (val/wer)  |

## 6. Оцінка на тестовій вибірці

### 6.1 Загальні результати

Оцінка виконувалась на **5 514 зразках** (28 зразків не вдалось обробити через помилки завантаження аудіо).

| Метрика | Значення |
|---------|:--------:|
| **WER** | **26.08%** |
| **CER** | **11.28%** |

### 6.2 Приклади транскрипцій

Нижче наведено репрезентативні приклади з тестової вибірки:

| ID | WER | CER | Референція | Гіпотеза |

| toronto_157_0 | 0.000 | 0.000 | слава ісу ви сі дивите програму грати песик... | слава ісу ви сі дивите програму грати песик... ✓ |
| toronto_157_1 | 0.056 | 0.008 | **купол** верховної ради впав... | **купл** верховної ради впав... |
| toronto_157_3 | 0.071 | 0.033 | теракт вона **планувала** влаштувати | теракт вона **повинувала** влаштувати |
| toronto_157_4 | 0.000 | 0.000 | що надія савченко особисто планувала | що надія савченко особисто планувала ✓ |

Як видно з прикладів, модель добре справляється з простими реченнями (WER=0), але допускає помилки при нестандартних словах, оказіоналізмах та швидкому мовленні.

### 6.3 Аналіз помилок

Основні типи помилок:

1. **Фонетична плутанина** — слова зі схожим звучанням підміняються (`купол → купл`, `добивають → добувають`). Це типова проблема ASR-систем на близькозвучних словах.
2. **Втрата частин слова або злиття** — модель іноді "ковтає" склади у довгих словах або зливає коротке слово з наступним (`про те → проти`).
3. **Галюцинації у складних сегментах** — на уривках зі швидким мовленням, шумом або незвичайним словником модель генерує нерелевантні токени (`кров вогонь агонія → крову гоня гоня`).

## 7. Висновки

1. Обрано модель `openai/whisper-small` як оптимальний компроміс між якістю та ефективністю.
2. Реалізовано навчальну логіку засобами **PyTorch Lightning** з заморозкою енкодера, scheduler'ом з warmup та ранньою зупинкою.
3. Модель дотренована на вибірці Toronto Dataset (11 484 зразки для навчання).
4. Тестова вибірка (`test_lines`) суворо не використовувалась під час навчання.
5. На тестовій вибірці досягнуто **WER = 26.08%** та **CER = 11.28%**.

Отримані результати є цілком прийнятними для відносно невеликої кількості даних. Для подальшого покращення можна розглянути: використання більшої моделі (`whisper-medium`), аугментацію аудіо даних, розморозку частини шарів енкодера в пізніх епохах, або доповнення тренувальних даних іншими українськими датасетами (наприклад, Common Voice uk).
